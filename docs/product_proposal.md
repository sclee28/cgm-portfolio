# 웰다 서비스 고도화 제안서

> **관점:** 이 제안서는 외부 연구자로서 공개 자료(앱 리뷰, 경쟁사 분석, CGMacros 분석 결과)를  
> 바탕으로 작성했다. 내부 데이터 없이 작성한 것이므로 실제 우선순위는 내부 지표로 검증 필요.

---

## 1. 현황 진단 — CGM 앱의 공통적 한계

| 현재 기능 | 사용자 관점의 한계 |
|---|---|
| TIR / GMI 하루 요약 | "혈당이 73%였다" → "그래서 어쩌란 거지?" |
| 식사 로그 | "탄수화물 60g" → "그게 많은 건지 적은 건지 모르겠다" |
| 트렌드 그래프 | 예쁜 그래프 → 행동으로 연결되지 않음 |
| 일반 권고사항 | "채소 더 먹으세요" → 개인에게 맞지 않는 조언 |

**핵심 문제:** 지표는 풍부하지만 **"나는 다음에 뭘 해야 하나?"**에 답하지 못한다.

---

## 2. 경쟁사 벤치마크

| 서비스 | 핵심 차별점 | 한계 |
|---|---|---|
| Dexcom Clarity | TIR·GMI 표준 대시보드 | 의료진 중심, 소비자 UX 약함 |
| Abbott LibreView | Libre 기기와 완벽 연동 | 기기 종속, 식사 분석 없음 |
| Levels Health (US) | "Metabolic Fitness Score" + 식사 점수화 | 미국만, CGM 기기 필요 |
| NutriSense (US) | CGM + 영양사 코칭 결합 | 고가, 확장성 낮음 |

**웰다의 기회:** 한국 시장에서 개인화 식사 분석 + 코칭 자동화를 결합한 서비스가 없다.

---

## 3. 제안 1: 식사 단위 혈당 반응 리포트 (PPGE·iAUC 기반)

### 현재 → 제안

```
현재: "오늘 TIR 72%, 평균 혈당 112 mg/dL"

제안: "오늘 점심의 혈당 반응: +52 mg/dL (고반응)
      주요 원인: 탄수화물 78g (평소 대비 +20g)
      같은 식사 유형의 다른 사용자 평균: +35 mg/dL
      개선 제안: 탄수화물을 15g 줄이거나 식전 10분 걷기를 추가해보세요"
```

### 구현 방안

1. 각 식사 후 iAUC 자동 계산 (실시간 CGM 스트리밍 + `compute_iauc()`)
2. XGBoost 예측 + SHAP → 주요 원인 피처 추출
3. `shap_to_coaching_message()` 로 코칭 메시지 자동 생성
4. 비슷한 세그먼트 사용자와 비교 (개인 vs 세그먼트 평균)

### 기대 효과

- 사용자의 즉각적 행동 변화 유도 → DAU 및 식사 로그 완성도 향상
- 코칭 메시지 클릭률 → 서비스 효과 A/B 측정 가능

---

## 4. 제안 2: 대사 유형 세그멘테이션 + 맞춤 코칭 트랙

### 세그먼트 정의 (CGMacros 분석 결과 기반)

| 세그먼트 | 특징 | 코칭 방향 |
|---|---|---|
| Stable | TIR >80%, CV% <30% | 유지 강화 + 고급 지표 제공 |
| Moderate | TIR 60~80%, 일부 고반응 식사 | 식사 최적화 + 주 1회 패턴 리뷰 |
| At-risk | TIR <60%, CV% >40% | 전문 코칭 연결 + 일일 체크인 |

### 코칭 트랙 예시

```
[Stable 사용자]
매주 월요일: "지난주 베스트 식사 TOP 3 → 이번 주에도 시도해보세요"

[At-risk 사용자]
매일 아침: "어제 가장 높은 혈당 반응 식사: 저녁 (iAUC 4,200)
            오늘 저녁 실험 제안: 같은 메뉴에서 밥 반 공기 줄이기"
```

### 구현 방안

1. `03_segmentation.ipynb` 파이프라인 → 신규 사용자 14일 후 자동 분류
2. 세그먼트 × 식사 유형 × 지표 임계값 → 룰 기반 코칭 엔진
3. 세그먼트 이동 감지 (Stable → Moderate 시 알림)

---

## 5. 제안 3: KPI 체계 재설계

### 현재 예상 KPI (일반적)

- DAU / MAU
- 앱 내 체류 시간

### 제안 KPI (CGM 서비스 특화)

| KPI | 정의 | 측정 방법 |
|---|---|---|
| TIR Improvement Rate | 4주 후 TIR 변화량 (% point) | 사전-사후 비교 |
| Meal Insight CTR | 코칭 메시지 클릭률 | 앱 이벤트 로그 |
| High-response Meal Reduction | 주당 PPGE >60 식사 횟수 감소율 | CGM + 식사 로그 |
| Segment Upgrade Rate | Moderate/At-risk → 상위 세그먼트 이동 비율 | 14일 단위 재분류 |
| Logging Consistency | 식사 로그 완성도 (% of meals logged) | 앱 로그 |

**왜 이 KPI인가:**  
DAU/체류시간은 "앱을 많이 보는 것"을 측정하지만 "혈당이 실제로 개선됐는지"를 놓친다.  
TIR Improvement Rate는 웰다의 핵심 가치(건강 개선)를 직접 측정한다.

---

## 6. 제안 4: 지속 사용성 설계

### CGM 이탈 패턴 (일반적 문제)

- 2~4주 후 "숫자 피로": 매일 혈당 확인이 부담  
- 식사 로그 귀찮음: 사진 촬영 → 수동 수정 과정  
- "그래서 뭐가 달라졌는지 모르겠음": 개선 체감 없음

### 지속성 향상 방안

1. **주간 요약 대시보드:** "이번 주 당신의 3가지 발견" (데이터 → 인사이트 압축)
2. **점진적 목표 설정:** TIR +5%p 달성 시 다음 목표 자동 제시 (게이미피케이션)
3. **식사 로그 자동화:** 음식 사진 → AI 영양소 추정 정확도 향상 (사용자 수정 최소화)
4. **"이번 달 가장 좋아진 것":** 긍정 강화 메시지로 이탈 방지

## 7. 실행 우선순위 제안

```
P0 (즉시):   식사 단위 PPGE·iAUC 리포트 + SHAP 코칭 메시지
             → 이미 CGMacros 분석으로 기술 검증 완료, 앱 연동만 필요

P1 (1~2개월): 대사 유형 세그멘테이션 + 세그먼트별 코칭 트랙
              → K-Means 파이프라인 완성, 실서비스 데이터로 재검증 필요

P2 (3~6개월): KPI 체계 재설계 + TIR Improvement A/B 테스트
              → 서비스 로그 기반 인과 검증 필요

P3 (6개월+):  CF 추천 시스템 (수천 명 interaction 확보 후)
              MHS 가중치 임상 학습 (의료 결과 데이터 연결 후)
```
