## 담당업무 2: 예측/추천 모델 개발 및 검증 — 커버리지 70%

**[JD 원문]**  
시계열 분석·예측 모델·추천 모델을 활용해 맞춤형 건강관리 서비스 개발;  
모델 성능 평가 지표 설계·오프라인 검증·온라인 실험으로 품질 개선;  
분석 결과와 모델 산출값을 서비스 적용 가능한 형태로 구체화

| JD 세부 항목 | 구현 여부 | 포트폴리오 위치 |
|---|:---:|---|
| 시계열 분석 | ✅ | `01_eda.ipynb` — 식후 혈당 곡선·일중 패턴 |
| 예측 모델 | ✅ | `02_ppgr_model.ipynb` — XGBoost iAUC 예측 |
| 추천 모델 | ❌ CF 불가 | `product_proposal.md` — 룰 기반 추천 설계 |
| 평가 지표 설계 | ✅ | RMSE/MAE/R² + 피험자별 성능 분해 |
| 오프라인 검증 | ✅ | GroupKFold(n=5) + LOSO 비교 |
| 온라인 실험 | ❌ 실서비스 필요 | `methodology.md` — A/B 설계 |
| 서비스 적용 형태 | ✅ | SHAP Waterfall → "탄수화물이 혈당 반응 ↑" 메시지 |

**핵심 증명:**  
- 45명 소규모 데이터의 분산 최대화: GroupKFold로 subject-level 누수 방지  
- Population 모델(전체 학습) vs Personalized 모델(LOSO + 편향 보정) 정량 비교  
- SHAP `shap_to_coaching_message()` 함수로 모델 출력을 서비스 언어로 자동 변환

**한계 명시:**  
- R²가 낮을 수 있음 (45명, 개인 간 분산이 크기 때문) — 누수가 아닌 데이터 특성  
- CF 추천은 최소 수천 명 이상의 interaction matrix 필요 → 설계만 제시

## 이 포트폴리오가 의도적으로 포함하지 않은 것

| 항목 | 이유 |
|---|---|
| Collaborative Filtering 추천 | 45명으로 CF 학습 불가 (통계 신뢰 불가) |
| R² > 0.8 | 소규모 데이터에서 달성 시 누수 의심 |
| AGE 축적 추정 | CGM으로 직접 계산 불가 |
| "최신 SOTA 모델" | 실제 서비스 적합성보다 해석 가능성 우선 |
| 재무/매출 분석 | 외부인이 접근 불가한 내부 데이터 |
