# 한계 및 향후 방향 (Limitations & Future Work)

> 이 문서는 분석 결과의 한계를 솔직하게 기술한다.  
> 한계를 아는 것이 한계를 모르는 것보다 낫다.

---

## 1. 데이터 한계

### 1-1. 소규모 표본 (n=45)

- **문제:** 45명은 통계적 검정력이 낮고, 서브그룹 분석(성별·연령대별)에서  
  각 셀당 샘플이 10명 미만으로 줄어들어 결과가 불안정해진다.  
- **영향:** K-Means 클러스터 경계가 불안정할 수 있으며,  
  XGBoost R²가 낮게 나와도 모델 품질 문제가 아닌 데이터 분산 특성일 수 있다.  
- **향후:** 실서비스 사용자 수천 명 규모로 재검증 필요.

### 1-2. 코칭 서비스 로그 없음

- **문제:** 실서비스 앱 내 코칭 메시지 조회, 클릭, 행동 변화 로그가 없다.  
  "코칭 메시지 → 실제 행동 변화" 인과 고리를 검증할 수 없다.  
- **향후:** 실서비스 로그 확보 시 코칭 메시지 노출 A/B 테스트 설계  
  (`methodology.md` §5 참조).

### 1-3. 날짜 시프트 (365일 이동)

- CGMacros 데이터는 개인정보 보호를 위해 타임스탬프가 365일 이동됐다.  
- 계절성(여름/겨울 식이 패턴)이나 특정 이벤트 효과를 분석할 수 없다.

### 1-4. 식사 로그 자기보고 편향

- 식이 데이터(탄/단/지)는 피험자 자기보고. 실제 섭취량과 차이가 있을 수 있다.  
- 음식 사진 기반 AI 추정이라 음식 구성이 복잡할수록 오차 증가.

---

## 2. 모델 한계

### 2-1. iAUC 예측 R²의 낮은 가능성

- **문제:** 45명의 개인 간 혈당 반응 분산이 크고, GroupKFold로  
  개인 특성 누수를 철저히 막았기 때문에 R²가 0.2~0.4 수준일 수 있다.  
- **해석:** 이는 모델 결함이 아니라 개인화의 어려움 자체를 드러낸다.  
  Population 모델보다 Personalized 모델의 RMSE 개선폭이 핵심 지표다.

### 2-2. 추천 시스템 미구현

- 45명으로 Collaborative Filtering 학습은 통계적으로 불가하다.  
  (user-item matrix가 거의 비어 있음)  
- 룰 기반 추천(세그먼트 × 식사 유형 × 지표 임계값)으로 대체했으며,  
  실서비스에서는 수천 명 이상의 interaction 데이터로 CF 구현 가능.

### 2-3. 외부 검증(External Validation) 없음

- 동일 데이터셋 내 교차검증만 수행. 다른 모집단(아시안, 노인, 특정 질환군)에서  
  모델 성능이 유지되는지 확인되지 않았다.

## 3. 지표 한계

### 3-1. MHS (Metabolic Health Score) 가중치 미검증

- MHS 공식의 가중치(w1~w5)는 임상 결과와의 상관관계로 결정되어야 한다.  
- 45명 데이터로는 가중치를 통계적으로 추정할 수 없어 프레임워크만 제시했다.  
- **향후:** 실서비스 사용자 + 의료 결과(HbA1c 변화, 체중 감소) 연결 시 추정 가능.

### 3-2. Recovery Time NaN 비율

- 2시간 윈도우 내 혈당이 기저로 돌아오지 않으면 NaN으로 처리.  
- 고혈당 반응 식사에서 NaN 비율이 높아질 수 있으며, 이를 그대로 삭제하면  
  분석 대상이 "회복이 빠른 식사"로 편향될 수 있다.

## 4. 향후 발전 방향

### 단기 (실서비스 데이터 확보 시 즉시 적용 가능)

| 항목 | 설명 |
|---|---|
| PPGE 기반 A/B 테스트 | 개인화 식단 추천 vs 일반 추천의 TIR 개선 효과 검증 |
| 실시간 iAUC 예측 API | `train_final_model()` → FastAPI 서빙 |
| 코칭 메시지 클릭률 추적 | SHAP 설명 → 사용자 반응 피드백 루프 |
| 식사 패턴 이상 감지 | 전일 대비 iAUC 급증 시 푸시 알림 트리거 |

### 중기 (사용자 수백~수천 명 수준)

| 항목 | 설명 |
|---|---|
| Collaborative Filtering | 비슷한 대사 유형 사용자의 "좋은 식사" 추천 |
| MHS 가중치 학습 | 의료 결과(HbA1c, 체중)와의 회귀로 가중치 추정 |
| CGM 시계열 딥러닝 | LSTM/Transformer 기반 분 단위 혈당 예측 |
| 다기관 외부 검증 | 한국인 코호트로 지표 기준값 재보정 |

### 장기 (전략적 방향)

| 항목 | 설명 |
|---|---|
| 의료기기 규제 대응 | 혈당 예측을 진단 목적으로 확장 시 식약처 허가 경로 |
| 다모달 데이터 통합 | CGM + 수면 + 스트레스 + 월경 주기 통합 분석 |
| 연속 개인화 | 사용자 데이터가 쌓일수록 자동으로 모델 업데이트 |

## 5. "절대 하지 말 것" 체크리스트 (준수 여부)

| 항목 | 준수 여부 |
|---|:---:|
| "웰다 매출이 적다" 등 재무 진단 언급 | ✅ 하지 않음 |
| R² > 0.8 달성 주장 | ✅ 하지 않음 |
| AGE 축적 추정 | ✅ 하지 않음 |
| CF 추천 구현 완료로 제시 | ✅ 하지 않음 |
| 구현 안 한 것을 구현한 척 | ✅ 하지 않음 |
| "최신 SOTA" 표현 | ✅ 하지 않음 |
