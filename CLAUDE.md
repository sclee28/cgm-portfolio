## 1. 담당업무 × CGMacros 현실적 커버리지 진단

채용공고 담당업무 4개를 CGMacros 데이터로 어디까지 보여줄 수 있고,
어디가 한계인지 정직하게 진단한다.

### 담당업무 1: 데이터 분석 및 개인화 알고리즘 설계

```
[JD 원문]
- CGM 데이터, 식이/행동 데이터, 코칭 서비스 로그 등 바이오·행동 데이터 분석
- 혈당반응, 식습관 패턴, 사용자 세분화/프로파일링 기반 맞춤형 알고리즘 설계
- 사용자 상태와 맥락을 반영한 건강관리·다이어트 인사이트 로직 개발
```

| 세부 항목 | CGMacros로 가능? | 구현 계획 |
|-----------|:---:|-----------|
| CGM 데이터 분석 | ✅ | Dexcom+Libre 1분 간격 시계열 EDA, 그룹별 분포 |
| 식이 데이터 분석 | ✅ | 매크로영양소(탄/단/지/섬유) + 식사 타입별 혈당 반응 |
| 행동 데이터 분석 | ⚠️ 부분 | Fitbit 심박/MET 활용 가능, 앱 사용 로그는 없음 |
| 코칭 서비스 로그 | ❌ 없음 | → "코칭 룰 엔진 설계" 문서로 대체 |
| 혈당반응 패턴 분석 | ✅ | PPGE, Recovery Time, 식사별/시간대별 패턴 |
| 사용자 세분화 | ✅ | 지표 기반 K-Means 클러스터링 → 대사 유형 분류 |
| 개인화 알고리즘 | ✅ | Population vs Personalized 모델 비교 |
| 인사이트 로직 | ✅ | 세그먼트 × 지표 → 코칭 메시지 룰 엔진 |

**판정: 80% 커버 가능.** 코칭 로그만 없고, 나머지는 직접 구현 가능.

### 담당업무 2: 예측/추천 모델 개발 및 검증

```
[JD 원문]
- 시계열 분석, 예측 모델, 추천 모델을 활용해 맞춤형 건강관리 서비스 개발
- 모델 성능 평가 지표 설계, 오프라인 검증 및 온라인 실험으로 품질 개선
- 분석 결과와 모델 산출값을 서비스 적용 가능한 형태로 구체화
```

| 세부 항목 | CGMacros로 가능? | 구현 계획 |
|-----------|:---:|-----------|
| 시계열 분석 | ✅ | 식후 혈당 곡선 분석, 일중 패턴 |
| 예측 모델 | ✅ | XGBoost로 PPGR 예측 (iAUC/PPGE) |
| 추천 모델 | ⚠️ 설계만 | 45명으로 CF 학습 불가 → 룰 기반 추천 설계 |

| 온라인 실험 | ❌ 불가 | → A/B 테스트 설계 문서로 대체 |

## 2. 혈당 지표 레퍼런스

### 2-A. 기본 지표 (업계 표준 — 구현 필수)

이 지표들은 Dexcom Clarity, Abbott LibreView 등 기존 CGM 앱이
공통으로 제공하는 업계 표준이다. "이것도 모르면 탈락"하는 기본기.

#### TIR (Time in Range, 범위 내 시간 비율)

```
정의:    혈당이 목표 범위 내에 있는 시간의 비율
공식:    TIR = (70 ≤ glucose ≤ 140 인 측정 수) / 전체 측정 수
단위:    % (0~100)
기준값:  건강인 >80%, 당뇨 환자 >70% (국제 합의 가이드라인)
의미:    하루 혈당 안정성의 가장 대표적인 단일 요약 지표
웰다 활용: "오늘 혈당이 안정된 시간: 83% → 목표 달성!"
참고:    Battelino et al., Diabetes Care 2019 — 국제 CGM 합의 권고
         건강인은 70~140 기준 권장, 당뇨 환자는 70~180 사용
```

#### TAR (Time Above Range, 고혈당 시간 비율)

```
정의:    혈당이 상한 초과인 시간의 비율
공식:    TAR = (glucose > 140 인 측정 수) / 전체 측정 수
단위:    %
기준값:  건강인 <25%, 당뇨 <50%
의미:    고혈당 노출 정도 — 장기적 합병증 위험과 직결
웰다 활용: TIR과 쌍으로 표시 → "범위 이탈 17% → 어떤 식사 때문?"
```

#### TBR (Time Below Range, 저혈당 시간 비율)

```
정의:    혈당이 하한 미만인 시간의 비율
공식:    TBR = (glucose < 70 인 측정 수) / 전체 측정 수
단위:    %
기준값:  <4% (Level 1), <1% (Level 2: <54 mg/dL)
의미:    저혈당 위험도 — 즉각적 건강 위험
웰다 활용: 다이어트 앱에서 특히 중요 (과도한 식이 제한 → 저혈당 위험 경고)
```

#### GMI (Glucose Management Indicator)

```
정의:    CGM 평균 혈당으로부터 HbA1c를 추정하는 지표
공식:    GMI(%) = 3.31 + (0.02392 × mean_glucose_mg_dL)
단위:    % (HbA1c 스케일)
기준값:  <5.7% 정상, 5.7~6.4% 전당뇨, >6.4% 당뇨
의미:    채혈 없이 CGM만으로 장기 대사 건강 추정 가능
웰다 활용: "2주간 당신의 대사 건강 추정치: GMI 5.3% (정상 범위)"
참고:    Bergenstal et al., Diabetes Care 2018
         14일 이상 CGM 착용 시 신뢰도 높음
```

#### CV% (Coefficient of Variation, 변동계수)

```
정의:    혈당 변동성의 표준화된 척도
공식:    CV% = (표준편차 / 평균) × 100
단위:    %
기준값:  <36% 안정적 (국제 합의), <33% 매우 안정적
의미:    혈당의 "들쭉날쭉" 정도 — 평균이 같아도 CV가 다를 수 있음
웰다 활용: "혈당 안정도: 28% → 안정적인 편입니다"
참고:    Monnier et al., Diabetes Care 2017
         CV%가 높을수록 저혈당 발생 위험 증가
```

### 2-B. 확장 지표 (차별화 — 포트폴리오 어필)

기존 CGM 앱이 제공하지 않거나 제한적으로 제공하는 지표.
웰다가 경쟁사 대비 차별화할 수 있는 "새로운 가치" 영역.

#### ① PPGE (Post-Prandial Glucose Excursion, 식후 혈당 상승폭)

```
정의:    식후 최고 혈당 - 식전 기저 혈당
공식:    PPGE = max(glucose[meal : meal+120min]) - glucose[meal-5min]
단위:    mg/dL
기준값:  <30 mg/dL 이상적, >60 mg/dL 고반응
의미:    "이 음식이 내 혈당을 얼마나 올렸는가" — 가장 직관적인 식사 평가

왜 차별화인가:
  - 기존 앱: TIR/GMI 같은 "하루 요약"만 → 어떤 식사가 문제인지 모름
  - PPGE: 식사 단위로 혈당 영향 정량화
  - 사용자에게 즉각적 행동 변화 유도 가능

웰다 활용:
  "오늘 점심의 혈당 상승: +52 mg/dL"
  "같은 메뉴를 먹은 비슷한 사용자 평균: +35 mg/dL"
  "식전 10분 걷기를 추가하면 -15 mg/dL 효과 예상"

참고:    IDF Guideline — 식후 2시간 혈당 <140 mg/dL 권장
```

#### ② iAUC (incremental Area Under the Curve, 증분 곡선하 면적)

```
정의:    식전 기저 혈당 위로 올라간 영역의 면적 (사다리꼴 적분)
공식:    iAUC = ∫(glucose(t) - baseline) dt, [meal, meal+120min]
         기저 아래로 내려간 부분은 0으로 처리 (양수만 적분)
단위:    mg/dL × min
기준값:  절대 기준보다 개인 내 변화 추적이 핵심

왜 차별화인가:
  - PPGE는 최고점만 봄
  - iAUC는 "총 혈당 노출량" 측정 → 더 정밀한 식사 평가
  - 예: A식사(피크 160, 30분 후 정상화) vs B식사(피크 155, 2시간 유지)
    → PPGE는 비슷, iAUC는 B가 훨씬 큼 → B가 실제로 더 나쁜 식사

웰다 활용:
  PPGE + iAUC 함께 → "최고점은 비슷했지만 오래 높았던 식사는 저녁"

참고:    Zeevi et al., Cell 2015 (개인화 영양 연구에서 iAUC 사용)
         CGMacros 공식 예제도 iAUC를 XGBoost 타겟으로 사용
```

#### ③ BGI (Blood Glucose Index, 혈당 위험 지수)

```
정의:    혈당의 비대칭적 위험을 정량화하는 지수
         고혈당 위험(HBGI)과 저혈당 위험(LBGI)을 분리

공식:
  1) f(BG) = 1.509 × (ln(BG)^1.084 - 5.381)
  2) rl(BG) = min(f(BG), 0)² × 10       ← 저혈당 성분
  3) rh(BG) = max(f(BG), 0)² × 10       ← 고혈당 성분
  4) LBGI = mean(rl), HBGI = mean(rh)

단위:    무단위 지수
기준값:  LBGI <2.5 낮은 위험, HBGI <5 낮은 위험

왜 차별화인가:
  - CV%는 고혈당/저혈당 구분 불가
  - BGI는 비대칭 분리 → 다이어트 중 저혈당 위험 특이적 감지 가능

웰다 활용:
  "고혈당 위험: 2.1 (낮음) | 저혈당 위험: 4.8 (주의)"
  "최근 3일 저혈당 위험 ↑ → 아침 식사를 거르지 마세요"

참고:    Kovatchev et al., Diabetes Technology & Therapeutics 2006
```

#### ④ Recovery Time (식후 혈당 정상화 시간)

```
정의:    식후 혈당이 식전 기저 혈당 수준으로 돌아오는 데 걸리는 시간
공식:    식후 혈당이 (기저값 ± threshold)에 처음 진입하는 시점 - 식사 시점
         threshold 기본값: 5 mg/dL
단위:    분 (min)
기준값:  <90분 양호, 90~150분 보통, >150분 인슐린 저항 의심

왜 차별화인가:
  - 인슐린 민감성의 간접 지표 — 비침습적 추정의 유일한 방법 중 하나
  - 같은 PPGE라도 빠르게 정상화 = 건강, 느리면 인슐린 저항
  - 시간 경과에 따른 개선 추적 가능

웰다 활용:
  "이 식사 후 정상화까지 135분"
  "운동 추가 시 평균 95분 (vs 미운동 140분)"

참고:    건강인 60~120분, 2형 당뇨 180분+
```

#### ⑤ MAGE (Mean Amplitude of Glycemic Excursions)

```
정의:    1SD 이상의 혈당 변동만 추출하여 평균한 값
공식:
  1) 혈당 시계열에서 극값(peak, nadir) 추출
  2) 연속 극값 간 차이 > 1SD인 것만 필터링
  3) 해당 차이들의 평균
단위:    mg/dL
기준값:  건강인 <40 mg/dL, 당뇨 <80 mg/dL

왜 차별화인가:
  - CV%는 잔잔한 변동 + 큰 변동 혼합 계산
  - MAGE는 "의미 있게 큰 변동"만 측정 → 임상적으로 더 의미 있음
  - 구현 자체가 기술력 증명 (알고리즘 난이도 중상)

웰다 활용:
  "오늘 큰 혈당 변동 3회, 평균 폭 55 mg/dL"

참고:    Service et al., Diabetes 1970
구현상태: ⚠️ 시간 여유 시 구현
```

#### ⑥ MHS (Metabolic Health Score, 대사 건강 점수)

```
정의:    여러 지표를 가중 합산, 0~100점으로 변환한 복합 점수
공식(제안):
  MHS = w1 × norm(TIR) + w2 × norm(1-CV%/100) + w3 × norm(1-PPGE/max)
      + w4 × norm(nocturnal_stability) + w5 × norm(1-recovery/max)

왜 차별화인가:
  - "오늘 82점" 한 마디로 대사 건강 요약 → 최고의 소비자 친화 지표
```

- ❌ "최신 SOTA" 표현
